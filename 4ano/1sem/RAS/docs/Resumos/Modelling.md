
A modelação é parte integrante da engenharia, envolvendo abstração, simplificação e formalização. Implica selecionar conceitos e abstrações apropriados para construir um modelo que representa uma realidade específica para um dado propósito. Os pontos-chave incluem:

- **Propósito:** Um modelo simplifica a realidade, focando-se nos aspetos relevantes para os objetivos do modelador;
- **Abstração:** Remove detalhes desnecessários para destacar elementos essenciais;
- **Utilidade:** Os modelos são alternativas económicas a objetos ou processos do mundo real para análise e raciocínio.

As características dos modelos incluem:
- **Abstração:** Representação reduzida do sistema;
- **Compreensibilidade:** Pontos de vista simplificados para facilitar a compreensão;
- **Precisão:** Representação fiel das propriedades-chave;
- **Raciocínio:** Permite análise e previsões sobre o sistema;
- **Eficiência de Custos:** Mais barato de criar e analisar do que o sistema real.

## Características dos Modelos

Os modelos são classificados em três dimensões:

- **1. Forma:**
	- **Modelos Físicos:** Representação material, frequentemente versões em escala de objetos do mundo real;
	- **Modelos Simbólicos:** Abstrações lógicas ou matemáticas, como diagramas UML ou equações.
- **2. Representatividade:**
	- **Modelos Descritivos:** Descrevem sistemas ou fenómenos existentes (por exemplo, modelos meteorológicos);
	- **Modelos Prescritivos:** Definem como os sistemas devem comportar-se ou ser construídos, comummente usados em engenharia progressiva.
- **3. Perspetiva:**
	- **Modelos Estruturais:** Representam os componentes estáticos e a arquitetura de um sistema (por exemplo, Diagramas de Classes UML);
	- **Modelos Comportamentais:** Enfatizam aspetos dinâmicos, funcionais e temporais do sistema (por exemplo, diagramas de estados, diagramas de fluxo de dados).

## Modelos para Requisitos

Os modelos desempenham um papel crítico durante a fase de análise da engenharia de requisitos, fornecendo uma forma estruturada de documentar os requisitos do sistema. A UML (Unified Modelling Language) é amplamente utilizada na indústria, com diagramas específicos adequados a diferentes necessidades de modelação:

- **1. Modelos de Domínio:**
	- Capturam o vocabulário, conceitos e âmbito do domínio de um sistema;
	- Destacam elementos comuns entre sistemas relacionados.
- **2. Modelos de Casos de Uso:**
	- Definem as fronteiras do sistema e funcionalidades disponíveis aos utilizadores;
	- Consistem em atores (utilizadores ou sistemas) e casos de uso (funcionalidades específicas).
- **3. Modelos de Classes:**
	- Descrevem componentes do sistema, os seus atributos e relações;
	- Cruciais para o desenvolvimento de software orientado a objetos.
- **4. Modelos de Sequência:**
	- Representam a comunicação entre objetos ou componentes;
	- Mostram o fluxo de mensagens e as interações dinâmicas dentro do sistema.
- **5. Modelos de Estados:**
	- Representam os estados e transições de componentes do sistema ao longo do tempo;
	- Úteis para modelar comportamentos dinâmicos e condições temporais.
- **6. Modelos de Atividades:**
	- Ilustram o fluxo de controlo entre atividades num processo;
	- Úteis para modelação de processos de negócio e sistemas com fluxos de trabalho complexos.
